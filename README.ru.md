# Universo Platformo Laravel

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![PHP Version](https://img.shields.io/badge/PHP-8.2%2B-blue.svg)](https://www.php.net/)
[![Laravel](https://img.shields.io/badge/Laravel-11.x-red.svg)](https://laravel.com/)

**Реализация Universo Platformo / Universo MMOOMM / Universo Kiberplano на базе Laravel и связанных технологий PHP.**

> **Примечание:** Это реализация Universo Platformo на Laravel/PHP. Для версии на React/TypeScript см. [universo-platformo-react](https://github.com/teknokomo/universo-platformo-react).

Universo Platformo Laravel - это комплексная платформа для создания многопользовательских 3D/AR/VR-приложений, метавселенных и интерактивных цифровых миров. Эта реализация использует надёжную экосистему Laravel и зрелый инструментарий PHP для обеспечения масштабируемой и поддерживаемой бэкенд-инфраструктуры с современным подходом к фронтенду.

**Universo Platformo служит основой для реализации Universo Kiberplano** - глобальной системы планирования и исполнения, которая объединяет планы, задачи и ресурсы, управляя при этом роботами. Эта система направлена на создание комплексной структуры для всемирной координации усилий, оптимизации распределения ресурсов и обеспечения эффективной автоматизации через робототехнические системы, всё в рамках единой цифровой среды.

## Видение и цель

Наш замечательный проект, который поможет создать глобальную технокомизацию и спасти человечество от окончательного порабощения и полного уничтожения, в настоящее время находится на ранней стадии разработки. Мы реализуем версию Universo Platformo на базе Laravel, которая послужит основой для создания интерактивных 3D/AR/VR-приложений с надёжностью и масштабируемостью корпоративного уровня.

Более подробно обо всём этом написано в "Книге Будущего" и различных других наших материалах, большинство из которых пока плохо структурированы и не на английском языке, но прямо сейчас ведётся работа по созданию новой подробной документации, которая будет представлена на многих языках.

```
universo-platformo-laravel/

│   │   └── base/                  # Базовая реализация
│   ├── metaverses-frt/            # Пакет фронтенда метавселенных
│   │   └── base/                  # Базовая реализация
│   ├── metaverses-srv/            # Пакет бэкенда метавселенных
│   │   └── base/                  # Базовая реализация
│   └── [другие пакеты]/           # Дополнительные пакеты функциональности
├── app/                           # Ядро приложения Laravel
├── config/                        # Файлы конфигурации
├── database/                      # Миграции и наполнители базы данных
├── routes/                        # Определения маршрутов
├── resources/                     # Представления, ресурсы и локализация
├── storage/                       # Хранилище приложения
├── tests/                         # Наборы тестов
├── .env.example                   # Шаблон конфигурации окружения
├── composer.json                  # PHP зависимости
├── artisan                        # Laravel CLI
└── README.md                      # Этот файл
```

### Соглашение об именовании пакетов

Пакеты следуют последовательному шаблону именования:

- **Пакеты фронтенда**: `{имя-функции}-frt` (например, `clusters-frt`, `auth-frt`)
- **Пакеты бэкенда**: `{имя-функции}-srv` (например, `clusters-srv`, `auth-srv`)

Каждый пакет содержит подкаталог `base/` для своей базовой реализации, что позволяет в будущем создавать альтернативные реализации при необходимости.

## Основные функциональные области

### Управление кластерами
Трёхуровневая система сущностей для организации ресурсов:
- **Кластеры**: Организационные единицы верхнего уровня
- **Домены**: Группировки среднего уровня внутри кластеров
- **Ресурсы**: Отдельные элементы внутри доменов

### Метавселенные
Создание и управление виртуальными мирами:
- **Метавселенные**: Экземпляры виртуальных миров
- **Секции**: Области внутри метавселенных
- **Сущности**: Объекты и компоненты внутри секций

### Уники (юниты/персонажи)
Созданные пользователем сущности с расширенными иерархиями:
- Управление персонажами
- Системы инвентаря
- Отслеживание прогресса

### Пространства и холсты
Визуальное программирование и создание сцен:
- Системы графов на основе узлов
- Интеграция с LangChain
- Узлы UPDL (Universal Platform Description Language)
- Инструменты композиции сцен

### Аутентификация и профили
Управление пользователями и идентификация:
- OAuth2 аутентификация через Laravel Passport
- Профили и предпочтения пользователей
- Контроль доступа на основе ролей
- Интеграция с Supabase

### Система публикации
Развёртывание и совместное использование контента:
- Рабочие процессы публикации проектов
- Контроль версий
- Управление доступом

## Начало работы

### Предварительные требования

- PHP 8.2 или выше
- Composer 2.x
- MySQL/PostgreSQL (или учётная запись Supabase)
- Node.js и NPM (для ресурсов фронтенда)

### Установка

1. **Клонируйте репозиторий**

```bash
git clone https://github.com/teknokomo/universo-platformo-laravel.git
cd universo-platformo-laravel
```

2. **Установите PHP зависимости**

```bash
composer install
```

3. **Настройте переменные окружения**

```bash
cp .env.example .env
php artisan key:generate
```

4. **Настройте подключение к базе данных**

Отредактируйте файл `.env` с вашими учётными данными Supabase:

```env
DB_CONNECTION=pgsql
DB_HOST=your-project.supabase.co
DB_PORT=5432
DB_DATABASE=postgres
DB_USERNAME=postgres
DB_PASSWORD=your-password
```

5. **Запустите миграции**

```bash
php artisan migrate
```

6. **Установите Laravel Passport**

```bash
php artisan passport:install
```

7. **Запустите сервер разработки**

```bash
php artisan serve
```

Приложение будет доступно по адресу `http://localhost:8000`

### Рабочий процесс разработки

1. **Создайте новый пакет**
   - Создайте каталог в `packages/` согласно соглашению об именовании
   - Добавьте подкаталог `base/` для реализации
   - Создайте `composer.json` пакета при необходимости
   - Зарегистрируйте пакет в корневом `composer.json`

2. **Добавьте двуязычную документацию**
   - Создайте `README.md` на английском
   - Создайте `README.ru.md` на русском (идентичная структура)
   - Синхронизируйте количество строк и разделы

3. **Следуйте лучшим практикам Laravel**
   - Используйте правильную MVC структуру
   - Реализуйте сервисный слой для бизнес-логики
   - Пишите тесты для новых функций
   - Следуйте стандартам кодирования PSR-12

## Кросс-платформенные реализации

Universo Platformo разрабатывается на нескольких технологических стеках:

- **Laravel**: Этот репозиторий реализует Universo Platformo на Laravel и PHP (текущий)
- **React**: [Universo Platformo React](https://github.com/teknokomo/universo-platformo-react) - реализация на React и TypeScript
- **Godot**: [Universo Platformo Godot](https://github.com/teknokomo/universo-platformo-godot) - реализация на движке Godot
- **PlayCanvas**: [Universo Platformo Nebulo](https://github.com/teknokomo/universo-platformo-nebulo) - реализация на движке PlayCanvas

Каждая реализация разделяет одни и те же основные концепции и цели, используя преимущества своего технологического стека.
